# permission

Система контроля разрешений для pi-coding-agent. Контролирует доступ к файлам и инструментам на основе настраиваемых правил.

## Возможности

- **Allow/Deny правила** для чтения, записи и редактирования
- **Wildcard-поддержка** (`*`, `**`) для путей
- **Глобальная и локальная конфигурация**
- **Подтверждение пользователя** для ограничённых операций
- **Автообнаружение pi-coding-agent** — разрешено чтение файлов пакета, запись заблокирована
- **Категория-специфичные разрешения** — права на чтение не дают автоматически права на запись/редактирование

## Конфигурация

- Глобальный: `~/.pi/agent/permission-settings.json`
- Локальный: `.pi/permission-settings.json`

### Структура конфига

```json
{
  "rules": [
    { "path": "src/**", "category": "read", "type": "allow" },
    { "path": "*.env", "category": "write", "type": "deny" }
  ]
}
```

### Pi Package директория

Система автоматически обнаруживает директорию `node_modules/@mariozechner/pi-coding-agent` и разрешает чтение из неё. Запись и редактирование заблокированы. Это позволяет permission системе корректно работать с собственными файлами.

## Команды

- `/permissions` — показать статус
- `/permissions-reload` — перезагрузить конфиг
- `/permissions-clear-session` — очистить сессию

## Инструмент request_permission

Агент может заранее запросить разрешение:

```typescript
request_permission({
  path: ".env",
  action: "read",
  reason: "Need database config"
});
```
